# Задача 1. Введение в гетерогенную параллелизацию (Теоретическое задание (25 баллов))

## Что такое гетерогенная параллелизация? 

**Гетерогенная параллелизация**  — это подход к организации вычислений, при котором в рамках одной программы совместно используются разные типы вычислительных устройств, прежде всего CPU (центральный процессор) и GPU (графический процессор).
Каждое устройство применяется для тех задач, для которых оно наиболее эффективно:

* CPU выполняет управляющую, последовательную и логически сложную часть программы;
* GPU выполняет массовые параллельные вычисления над большими объёмами данных.

Таким образом, вычислительная нагрузка распределяется между разнородными архитектурами, что позволяет значительно повысить производительность по сравнению с использованием только CPU.

## Различия между параллельными вычислениями на CPU и GPU
1.Архитектурные различия

**CPU (Central Processing Unit)**:
- имеет небольшое количество мощных ядер (обычно от 4 до 16);
- оптимизирован для последовательного выполнения задач;
- хорошо справляется с ветвлениями, условиями, логикой управления;
- использует сложную иерархию кэш-памяти (L1, L2, L3).

**GPU (Graphics Processing Unit)**:
- содержит тысячи простых вычислительных ядер;
- ориентирован на массово-параллельные вычисления;
- работает по принципу SIMD (Single Instruction, Multiple Data) — одна инструкция выполняется сразу над множеством данных;
- состоит из потоковых мультипроцессоров (SM), каждый из которых обрабатывает множество потоков одновременно.

2. Модель выполнения
* На CPU параллелизм достигается за счёт небольшого числа потоков, часто с использованием OpenMP или потоков ОС.
* На GPU параллелизм реализуется за счёт тысяч потоков, объединённых в блоки и сетки (grid & blocks), которые выполняются аппаратно параллельно.

3. Работа с памятью
**CPU**:
- быстрый доступ к памяти;
- эффективная кэш-система;
- небольшое количество потоков обращаются к памяти.

**GPU**:
- несколько типов памяти:
- глобальная (медленная, но ёмкая),
- разделяемая (очень быстрая внутри блока),
- регистры (самая быстрая память);
- высокая пропускная способность памяти;
- эффективность сильно зависит от правильной организации доступа к данным.

## Преимущества гетерогенной параллелизации

1. Значительное ускорение вычислений
Использование GPU позволяет выполнять тысячи операций одновременно, что даёт кратный прирост производительности для вычислительно сложных задач.

2. Эффективное использование ресурсов системы
CPU и GPU работают параллельно, а не простаивают, каждый выполняя свою часть задачи.

3. Масштабируемость
Подход хорошо масштабируется при увеличении объёма данных: чем больше данных, тем эффективнее используется GPU.

4. Энергоэффективность
GPU выполняет большое количество операций с меньшими энергозатратами по сравнению с CPU при тех же вычислениях.

5. Гибкость программирования
С помощью технологий вроде CUDA разработчик может явно управлять распределением задач и памятью между CPU и GPU.



## Примеры реальных приложений гетерогенной параллелизации
1. Машинное обучение и нейронные сети
  - обучение глубоких нейронных сетей;
  - матричные операции и тензорные вычисления;
  - фреймворки: TensorFlow, PyTorch, CUDA.

2. Компьютерная графика и обработка изображений
  - рендеринг 3D-графики;
  - обработка видео и изображений;
  - трассировка лучей (ray tracing).

3. Научное и инженерное моделирование
  - моделирование физических процессов;
  - численные методы (метод конечных элементов, моделирование жидкостей);
  - климатическое и аэродинамическое моделирование.

4. Обработка больших данных
  - параллельная фильтрация и агрегация данных;
  - аналитика в реальном времени;
  - ускорение вычислений в финансовых и телекоммуникационных системах.

5. Криптография и вычислительная математика
  - хеширование;
  - перебор комбинаций;
  - оптимизационные задачи.

# Итог

Гетерогенная параллелизация представляет собой современный и эффективный подход к высокопроизводительным вычислениям, основанный на совместном использовании CPU и GPU.
Она позволяет:
- максимально использовать архитектурные особенности каждого устройства;
- значительно ускорять вычисления;
- решать задачи, которые невозможно эффективно выполнить только на CPU.

Именно поэтому гетерогенная параллелизация является ключевой технологией в современных вычислительных системах и активно применяется в научных, инженерных и прикладных задачах.
