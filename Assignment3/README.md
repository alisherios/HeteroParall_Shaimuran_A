Данный репозиторий содержит решения Assignment 3, посвящённого архитектуре GPU и оптимизации CUDA-программ.
В проекте реализованы CUDA-ядра для поэлементной обработки массивов и проведены измерения времени выполнения (CUDA events) для разных вариантов доступа к памяти и конфигураций запуска.

## Сборка и запуск

Каждое задание — отдельная программа. Компиляция и запуск (пример):

```bash
nvcc -O3 task_1.cu -o task_1
./task_1
```

Аналогично для `task_2`, `task_3`, `task_4`.

## Задание 1 (`task_1.cu`) — Global memory vs Shared memory

**Описание:**
- Реализована поэлементная обработка массива (умножение на константу) в двух вариантах:
  1) только глобальная память;
  2) с использованием разделяемой памяти.
- Выполняется сравнение времени для массива размером 1 000 000 элементов.

## Задание 2 (`task_2.cu`) — Влияние размера блока на сложение массивов

**Описание:**
- Реализовано поэлементное сложение двух массивов.
- Выполнены замеры времени для нескольких размеров блока потоков (минимум 3).

## Задание 3 (`task_3.cu`) — Коалесцированный и некоалесцированный доступ

**Описание:**
- Реализованы две версии обработки массива:
  - коалесцированный доступ (последовательные адреса);
  - некоалесцированный доступ (stride/перестановка индексов).
- Сравнивается время выполнения.

## Задание 4 (`task_4.cu`) — Оптимизация конфигурации сетки/блоков

**Описание:**
- Для одной из программ (сложение массивов) проведён перебор размеров блока.
- Сравнивается «неоптимальная» конфигурация (например, 64) и «оптимальная» (лучшее измеренное значение).
