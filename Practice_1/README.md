### Задание 1 (`task1.cpp`)

**Описание:**  
В этом задании реализована программа на C++ для работы с массивами. Программа создаёт массив из `N` элементов, заполняет его случайными числами в заданном диапазоне и находит минимальный и максимальный элементы. Реализованы два варианта обработки данных: последовательный и параллельный с использованием OpenMP.

**Цель задания:**  
- Ознакомиться с базовой работой с массивами в C++  
- Научиться использовать генерацию случайных чисел  
- Измерять время выполнения операций с помощью библиотеки `<chrono>`  
- Ознакомиться с параллельной обработкой данных с использованием OpenMP  

**Функционал программы:**  
1. Пользователь вводит размер массива `N`.  
2. Массив заполняется случайными числами в диапазоне `[RAND_MIN_VAL, RAND_MAX_VAL]`.  
3. Последовательный обход массива для поиска минимального и максимального значения с измерением времени выполнения.  
4. Параллельный обход массива с помощью директивы `#pragma omp parallel for reduction(min:min_par) reduction(max:max_par)` для нахождения минимума и максимума, также с измерением времени выполнения.  
5. Вывод результатов: минимальное и максимальное значения, а также время выполнения последовательного и параллельного вариантов.

### Задание 2 (`task2.cpp`)

**Описание:**  
В этом задании реализованы основные динамические структуры данных: односвязный список, стек и очередь. Программа демонстрирует работу с этими структурами как в последовательном, так и в параллельном режимах с использованием OpenMP.  

**Цель задания:**  
- Научиться реализовывать и использовать динамические структуры данных на C++  
- Познакомиться с базовыми операциями над структурами: добавление, удаление, поиск элементов  
- Ознакомиться с параллельной обработкой операций с динамическими структурами данных  
- Измерять и сравнивать производительность последовательной и параллельной реализации  

**Функционал программы:**  
1. Пользователь вводит количество элементов `N`, которые будут добавлены/удалены из структур.  
2. **Односвязный список (LinkedList)**:  
   - Добавление элементов (`add`)  
   - Удаление элементов (`remove`)  
   - Поиск элемента (`search`)  
   - Подсчёт количества элементов (`size`)  
   - Параллельное добавление и удаление с использованием OpenMP и мьютексов для потокобезопасности  
3. **Стек (Stack)**:  
   - Добавление элементов (`push`)  
   - Удаление элементов (`pop`)  
   - Проверка пустоты (`isEmpty`)  
   - Подсчёт количества элементов (`size`)  
   - Параллельное выполнение операций `push` и `pop`  
4. **Очередь (Queue)**:  
   - Добавление элементов в конец (`enqueue`)  
   - Удаление элементов из начала (`dequeue`)  
   - Проверка пустоты (`isEmpty`)  
   - Подсчёт количества элементов (`size`)  
   - Параллельное выполнение операций `enqueue` и `dequeue`  

**Измерение производительности:**  
- Время выполнения операций добавления и удаления измеряется с помощью `<chrono>`  
- Для параллельных операций рассчитывается ускорение (speedup) и эффективность (efficiency) с учётом числа потоков OpenMP

### Задание 3 (`task3.cpp`)

**Описание:**  
В этом задании реализуется работа с динамически выделенной памятью и вычисление среднего значения элементов массива. Программа демонстрирует как последовательный, так и параллельный подсчёт среднего с использованием OpenMP.  

**Цель задания:**  
- Научиться работать с динамической памятью и указателями в C++  
- Освоить базовые операции с массивами, выделенными в динамической памяти  
- Ознакомиться с параллельным вычислением с использованием OpenMP  
- На практике измерить и сравнить производительность последовательного и параллельного подхода  

**Функционал программы:**  
1. Пользователь вводит размер массива `N`.  
2. Создаётся динамический массив из `N` элементов.  
3. Массив заполняется случайными числами (от 0 до 9999).  
4. Вычисляется среднее значение элементов массива:  
   - **Последовательный вариант** — стандартный цикл `for`.  
   - **Параллельный вариант (OpenMP)** — цикл с директивой `#pragma omp parallel for reduction(+:sum)`.  
5. Измеряется время выполнения для обоих вариантов.  
6. Для параллельного варианта вычисляется ускорение (speedup) по сравнению с последовательным методом.  
7. В конце освобождается динамически выделенная память.
